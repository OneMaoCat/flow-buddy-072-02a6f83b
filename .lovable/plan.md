### 实现步骤

**1. 新建 `src/pages/DevExecution.tsx`**
- 顶部：返回按钮 + 标题 + 总体进度条（所有需求的加权进度）
- 主体：需求卡片列表，每个需求是一个可折叠卡片：
  - 卡片头：需求标题 + 状态标签（等待中/进行中/已完成） + 需求级进度条
  - 卡片体：该需求下的智能体列表（代码生成、测试编写、代码审查等），各自有独立进度和当前文件
  - 智能体之间有依赖关系（如审查Agent等待测试Agent完成）
- 底部：实时日志流（带时间戳，标注是哪个需求的哪个Agent）
- 模拟数据：3-4个并行需求，每个需求2-4个Agent，用 setInterval 模拟进度推进

**2. 更新路由 `src/App.tsx`**
- 添加 `/project/:id/dev` 路由指向 DevExecution

**3. 更新 `src/components/PlanFlow.tsx`**
- 确认开发后使用 `useNavigate` 跳转到 `/project/:id/dev`

### 技术细节
- 需求卡片使用 Collapsible 组件，默认展开进行中的需求
- 各需求进度独立模拟，不同需求错开启动时间
- Agent 状态色：等待(gray) / 执行中(blue+pulse) / 完成(green) / 出错(red)
- 使用 Progress 组件 + ScrollArea 组件
