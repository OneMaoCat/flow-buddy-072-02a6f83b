

## 验收面板增加产品预览体验功能

### 设计目标
在需求验收展开面板中嵌入产品预览入口，让验收者可以直接体验开发出来的界面效果，再决定通过或打回。

### 交互设计

```text
┌─ #3 支付模块集成  ██████████ 100%  🔍待验收 ──────────┐
│                                                        │
│  📋 Agent 执行详情（现有）                              │
│  ├─ 支付接入   ██████████ 100% ✅  payment.ts          │
│  └─ ...                                                │
│                                                        │
│  👁️ 产品预览                                           │
│  ┌──────────────────────────────────────────────────┐  │
│  │  ┌─────────────────────────────────┐             │  │
│  │  │                                 │  Desktop ○  │  │
│  │  │    嵌入式预览 iframe            │  Tablet  ○  │  │
│  │  │    (模拟开发成果展示)            │  Mobile  ○  │  │
│  │  │                                 │             │  │
│  │  └─────────────────────────────────┘             │  │
│  │  [🔗 新窗口打开完整预览]                          │  │
│  └──────────────────────────────────────────────────┘  │
│                                                        │
│  [✅ 通过]              [❌ 打回并备注]                 │
└────────────────────────────────────────────────────────┘
```

### 实现步骤

**1. 扩展 mock 数据** (`src/data/devExecutionMock.ts`)
- `Requirement` 接口新增 `previewPath?: string` 字段，表示该需求对应的预览路由（如 `/preview/payment-form`）
- 为每个需求模板分配一个合理的预览路径

**2. 内嵌预览组件** (新建 `src/components/RequirementPreview.tsx`)
- 接收 `previewPath` 和可选的设备类型
- 提供 Desktop / Tablet / Mobile 三种视口切换按钮
- 内部渲染一个带边框的模拟预览区域（当前无真实 iframe 目标，用占位 UI 模拟，显示对应需求名称和路由）
- 底部提供「新窗口打开」按钮，链接到 `/project/:id/preview`

**3. 集成到验收展开面板** (`src/pages/DevExecution.tsx`)
- 在 Agent 详情和验收操作按钮之间，插入预览区域
- 仅在 `review` 状态的需求展开时显示预览
- 预览区域可折叠，默认展开，高度约 300px

**4. 新窗口预览跳转**
- 「新窗口打开」按钮使用 `window.open` 打开已有的 `/project/:id/preview` 页面
