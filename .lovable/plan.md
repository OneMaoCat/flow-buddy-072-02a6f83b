

# 需求平台 + 异步开发 + 消息卡片验收

## 核心交互模型

```text
需求澄清平台（ProjectWorkspace）     异步开发（后台）      聊天消息卡片验收
┌──────────────────────┐          ┌──────────┐       ┌─────────────────────┐
│ 输入需求 → 澄清 →    │  提交开发  │ AI 自动   │ 完成  │ 推送消息卡片到聊天区  │
│ 生成文档 → 确认      │ ────────→ │ 编码+测试 │ ───→ │ 点开可看:             │
│                      │          │ 用户可离开 │      │ · 代码diff            │
│                      │          └──────────┘       │ · 产品预览            │
│                      │                             │ · 自测报告            │
│                      │                             │ [确认] → 发布到测试环境 │
└──────────────────────┘                             └─────────────────────┘
```

## 改动清单

### 1. 新建 `src/components/DevCompleteCard.tsx` — 开发完成消息卡片

聊天区内的可展开卡片组件，包含四个 Tab：
- **代码变更**：文件列表 + 模拟 diff（绿色加行/红色删行）
- **产品预览**：复用 RequirementPreview，内嵌设备切换
- **自测报告**：测试用例列表（通过/失败/通过率）
- **操作栏**：底部固定「确认无误，发布到测试环境」按钮 + 「打回修改」按钮

### 2. 修改 `src/components/PlanFlow.tsx` — 确认后的交互

- 确认需求后，不再跳转到 `/dev` 页面
- 改为在聊天区显示一条"已提交开发，AI 正在处理中…"的状态消息
- 模拟 3-5 秒后，在聊天区推送 DevCompleteCard 消息卡片
- 卡片出现时播放 sonner toast 通知（如果用户还在页面）

### 3. 修改 `src/pages/ProjectWorkspace.tsx` — 状态管理

- 新增 `devCards` 状态数组，存储已完成的开发结果
- PlanFlow 确认后触发异步模拟，完成后将结果 push 到 devCards
- 聊天区渲染 devCards 中的 DevCompleteCard
- 用户点击「确认发布」后，触发 toast "已发布到测试环境"

### 4. 新建 `src/components/DevNotification.tsx` — 浏览器通知

- 封装 `Notification API`，开发完成时发送浏览器通知
- 用户不在页面时通过系统通知拉回
- 点击通知聚焦回 ProjectWorkspace 页面

### 5. 模拟数据

- 在 DevCompleteCard 中内置模拟 diff 数据（3-4 个文件的代码变更）
- 模拟测试报告（8-10 条测试用例，全部通过）
- 模拟预览路径

## 用户体验

- 需求确认后，用户可以继续在聊天区输入新需求或直接离开
- 开发完成时，聊天区自动出现一张完成卡片（类似飞书机器人推送）
- 卡片内可完成所有验收动作，无需跳转其他页面
- 确认后一键发布到测试环境
