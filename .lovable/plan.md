

# 概览 Tab 重构：AI 驱动的完整验收报告

## 核心理念

当前概览只是指标仪表盘 + 时间线，用户仍需切换 Tab 查看细节。重构为**一份完整的、可滚动的 AI 验收报告**，人类只需从上往下阅读，最后做出"通过/打回"决定。其他 Tab 保留作为深入查看的入口，但绝大多数情况下概览就够了。

## 报告结构（自上而下）

```text
┌─────────────────────────────────────────┐
│  📋 AI 验收结论（一句话 + 建议操作）       │
│  ✅ 建议发布 / ⚠️ 建议修改后再发布         │
├─────────────────────────────────────────┤
│  1. 任务背景                              │
│     原始需求 · AI 理解摘要 · 关联对话       │
├─────────────────────────────────────────┤
│  2. 开发过程（时间线，保留现有）             │
├─────────────────────────────────────────┤
│  3. 代码变更摘要                           │
│     AI 总结改了什么 + 文件列表 + 行数统计    │
├─────────────────────────────────────────┤
│  4. AI Code Review 结果                   │
│     评分 + 各模型摘要 + 关键发现（内联）     │
├─────────────────────────────────────────┤
│  5. 测试报告                              │
│     通过率 + 用例列表 + 失败项高亮           │
├─────────────────────────────────────────┤
│  6. 产品预览                              │
│     内嵌预览截图/iframe，可点击放大          │
├─────────────────────────────────────────┤
│  底部：通过 / 打回 按钮（固定在底部）        │
└─────────────────────────────────────────┘
```

## 改动方案

### 1. DevCompleteDetailPanel.tsx — 重写概览 TabsContent

将现有的"指标卡 + 时间线 + 关键发现"替换为完整报告，分为 6 个 Section 组件：

- **AI 验收结论**：根据测试全过/AI 评分/有无 critical 自动生成一句话结论和建议操作（"建议发布"/"有 1 项严重问题需修复"）
- **任务背景**：从 `result` 中读取 `requirementTitle`，新增 `sourceContext` 字段传入（原始 prompt + AI 摘要）
- **开发过程**：保留现有时间线，微调样式
- **代码变更摘要**：AI 风格的变更说明 + 文件列表（从 `result.files` 渲染）
- **Code Review 内联**：评分环 + 各模型一句话摘要 + 所有 findings 按严重度排列
- **测试报告**：通过率进度条 + 用例列表（直接复用 tests 数据）
- **产品预览**：内嵌 `RequirementPreview` 组件，限高，可点击跳转预览 Tab

### 2. DevCompleteCard 类型扩展

`DevCompleteResult` 增加可选字段：
- `sourceContext?: { userPrompt: string; aiSummary: string }`
- `aiChangeSummary?: string`（AI 对代码变更的自然语言总结）

在 `buildMockDevResult` 中填充 mock 数据。

### 3. 底部操作栏调整

将"通过/打回"按钮从面板底部移入概览报告的最底部（滚动到底才看到），同时在顶部结论区也放一组操作按钮，方便快速决策。

### 涉及文件

| 文件 | 改动 |
|------|------|
| `src/components/DevCompleteDetailPanel.tsx` | 重写概览 Tab 为完整验收报告 |
| `src/components/DevCompleteCard.tsx` | `DevCompleteResult` 增加 sourceContext、aiChangeSummary 字段 |
| `src/pages/ProjectWorkspace.tsx` | 构建 result 时填充新字段 |
