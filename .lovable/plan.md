

# 验收报告卡片信息密度 + AI 结论优化

## 当前问题

从截图可以清晰看到：

1. **指标卡片信息密度低** — 4 个卡片占据大量纵向空间，每个卡片只有一个数字 + 一行标签，2x2 网格浪费空间。综合评分卡里环形图太大，测试通过率卡仅一个数字+细进度条，代码变更和执行耗时更是空洞。
2. **AI 验收结论奇怪** — 顶部 banner 写着"AI 验收结论"但实际只是一行泛泛的文字（"发现 1 个严重问题，建议修复后再发布"），没有关键数据支撑，verdict 和下方指标卡片信息重复。

## 方案

### 1. AI 验收结论重构 — 合并指标为信息丰富的 banner

将 verdict banner 和 4 个指标卡片**合并为一个大型仪表板区域**：

- 左侧：verdict 状态（大 emoji + 结论语句 + 关键行动提示）
- 右侧：指标以紧凑行内形式排列（评分环 + 测试 + 变更 + 耗时），不再用独立卡片

这样一行就同时传达了结论和数据，信息密度大幅提升。

### 2. 取消 2x2 指标卡片，改为横向指标条

4 个指标改为**单行横排**的 stat 条，每个 stat 用 icon + 数字 + 标签紧凑排列，用竖线分隔。整体高度从 ~200px 缩减到 ~60px。

```text
┌─────────────────────────────────────────────────────┐
│ 🚨 发现 1 个严重问题                                  │
│    审查评分 88 · 测试 5/5 通过 · 3 文件 +49 -9 · 14s  │
│                                          [2 个待决策] │
└─────────────────────────────────────────────────────┘
```

### 3. AI Code Review 三列评分卡优化

reviewer 三列评分卡（GPT-5 / Gemini / Claude）目前只有 emoji + 分数 + 名字，增加一行 summary 摘要，并将 emoji 缩小、分数加大。
