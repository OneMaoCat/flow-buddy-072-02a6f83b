

## 在验收前增加 AI 测试 Agent 环节

### 设计思路

当前流程：所有 Agent 完成 → 直接进入 `review`（人工验收）。问题是人工验收前没有自动化质量把关。

新流程：所有业务 Agent 完成 → **AI 测试 Agent 自动运行** → 测试全部通过 → 才进入 `review`。

```text
需求状态流转：
  waiting → running → testing (AI测试中) → review → accepted/rejected
                         ↓ 失败
                      testing_failed → 自动修复重试 → testing → ...

┌─ #3 支付模块集成 ─────────────────────────────────────┐
│                                                        │
│  📋 Agent 执行详情                                     │
│  ├─ 支付接入   ██████████ 100% ✅                      │
│  ├─ UI 组件    ██████████ 100% ✅                      │
│  └─ 支付测试   ██████████ 100% ✅                      │
│                                                        │
│  🤖 AI 测试 Agent                                      │
│  ┌──────────────────────────────────────────────────┐  │
│  │  通过率 ██████████░ 11/12 (91%)   耗时 3.2s      │  │
│  │  ✅ 支付表单渲染测试              120ms           │  │
│  │  ✅ 金额输入验证                   89ms           │  │
│  │  ✅ 支付方式切换                  156ms           │  │
│  │  ❌ 异常金额边界处理              203ms           │  │
│  │  ...                                              │  │
│  │  ┌────────────────────────────────────────────┐   │  │
│  │  │ 🔄 AI 正在自动修复并重试 (第 1 次)...      │   │  │
│  │  └────────────────────────────────────────────┘   │  │
│  └──────────────────────────────────────────────────┘  │
│                                                        │
│  测试全部通过后 → 自动进入待验收状态                    │
└────────────────────────────────────────────────────────┘
```

### 实现步骤

**1. 扩展状态类型** (`src/data/devExecutionMock.ts`)
- `RequirementStatus` 新增 `"testing"` 状态
- `Requirement` 接口新增 `testResult?: { tests: TestItem[]; retryCount: number }` 字段
- `TestItem` 类型：`{ name, status, duration }`，每个需求根据其 agents 自动生成 3-6 个测试用例

**2. 测试模拟逻辑** (`src/pages/DevExecution.tsx`)
- 需求所有业务 Agent 完成 (`done`) → 状态先变为 `"testing"` 而非直接 `"review"`
- `testing` 状态下模拟测试用例逐个执行（类似现有 Agent 进度动画）
- 首次运行约 15% 概率失败 → 自动触发修复重试（最多 3 次），重试时全部通过
- 测试全部通过 → 自动转为 `"review"`；日志输出测试结果

**3. 测试结果 UI** (`src/pages/DevExecution.tsx`)
- 在展开面板中 Agent 详情下方显示「AI 测试 Agent」区块
- 显示测试通过率进度条、用例列表（通过/失败/运行中状态图标）、耗时
- `testing` 状态时显示动态进度；完成后显示最终结果
- 修复重试中显示「AI 正在自动修复并重试（第 N 次）」提示

**4. 统计与筛选更新**
- 统计栏新增「测试中」计数卡片
- 筛选 Tab 新增「测试中」选项
- 需求行状态图标和标签增加 `testing` 对应的样式（蓝紫色 + TestTube2 图标）
