

# 验收报告优化：叙事式流畅阅读体验

## 问题

当前 6 个 Section 全部展开，信息密集，阅读压力大。需要一种更舒展、像阅读文档一样自然的体验。

## 方案：叙事式报告 + 智能折叠

核心思路：**像读一封邮件，而不是看仪表盘**。

### 关键改动

1. **智能默认折叠**：一切正常的 Section 默认折叠，只显示一行绿色 ✓ 摘要（如「✓ 10/10 测试全部通过」）。有问题的 Section 才自动展开。大多数场景下报告很短。

2. **去掉编号圆圈和紧凑边框**，改为宽松的分隔线分区，更大的字号和行高，更多留白（py-4 → py-6），阅读像浏览文档。

3. **Section 折叠态优化**：折叠时不是完全隐藏，而是显示一行内联摘要：
   - 任务背景：显示需求标题
   - 代码变更：「4 个文件 · +84 -8 行」
   - AI Review：「88 分 · 无严重问题」或「⚠ 2 项警告」
   - 测试：「✓ 10/10 全部通过」或「✗ 2 项失败」
   - 预览：缩略图

4. **verdict banner 更大更醒目**，作为报告的"封面"，增加更多间距。

5. **Section header 改为文字链接风格**，点击展开/折叠，去掉卡片式背景，更轻量。

### 视觉效果

```text
───────────────────────────────
  ✅ AI 验收结论
  审查通过，综合评分 88 分，可以放心发布
  [发布]
───────────────────────────────

  任务背景                    ✓ 已理解
  "实现用户登录页面……"
  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─

  开发过程 · 5.2s             ✓ 完成
  （折叠，点击展开时间线）
  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─

  代码变更                    4 文件 · +84 -8
  （折叠，点击展开文件列表）
  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─

  AI Code Review              88 分 ✓
  （折叠，点击展开详情）
  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─

  测试报告                    ✓ 10/10
  （折叠）
  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─

  产品预览                    [展开]
  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─

  [确认发布到测试环境]
───────────────────────────────
```

当有问题时，对应 Section 自动展开显示完整内容 + 底部出现 Q&A 决策面板。
