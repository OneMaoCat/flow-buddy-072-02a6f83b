

## Plan: 大规模需求执行的仪表盘视图

当前逐条展开的卡片列表无法承载数百上千个需求。改为「概览仪表盘 + 筛选列表 + 详情展开」三层结构。

### 实现步骤

**1. 扩充 mock 数据到 50+ 个需求**
- 用模板批量生成不同类别的需求（用户模块、支付、通知、报表、权限等），每个 2-4 个 Agent
- 模拟并发上限提升到 8 个需求同时 running

**2. 顶部概览统计区**
- 4 个统计卡片：总需求数、执行中、已完成、等待中（实时更新）
- 右侧总体进度条保留

**3. 筛选与搜索栏**
- 状态筛选 Tabs：全部 / 执行中 / 已完成 / 等待中 / 出错，每个 tab 显示计数
- 搜索输入框：按需求标题实时过滤
- 筛选后列表即时响应

**4. 紧凑行列表替代大卡片**
- 每个需求压缩为单行：序号 + 标题 + 进度条 + 百分比 + 状态图标
- 行高约 36px，无需展开即可扫视全局
- 点击某行展开显示该需求的 Agent 详情（保留现有 Agent 行结构）
- 同时只展开一个需求（Accordion 模式）
- 已完成的需求默认收起且降低透明度

**5. 日志面板改为可折叠**
- 默认折叠为一行（显示最新一条日志），点击展开完整日志区
