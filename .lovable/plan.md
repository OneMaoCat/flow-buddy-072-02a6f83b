

# 验收报告画布化改版

## 问题

当前概览报告是线性的折叠列表，信息密度高但视觉层次不够，阅读体验偏"文档"而非"仪表盘"。

## 方案

将概览报告改为**画布式布局**：顶部是核心指标卡片区（评分、测试通过率、变更量），中部是可视化流水线时间轴，底部是详情区。信息通过空间、色彩、大小来传达，而非纯文字列表。

### 布局结构

```text
┌─────────────────────────────────────────┐
│  🚨 AI 验收结论（大字 verdict banner）    │
├──────────┬──────────┬──────────┬────────┤
│  综合评分  │  测试通过  │ 代码变更  │ 用时   │  ← 指标卡片
│   85 分   │  10/10   │ +142 -38 │ 4.2s  │
│  ██████░  │  ████████ │  5 files │       │
├──────────┴──────────┴──────────┴────────┤
│  ● ─── ● ─── ● ─── ● ─── ●             │  ← 横向时间轴
│  分支   编码  测试  审查  验收            │
├─────────────────────────────────────────┤
│  [AI Code Review 发现项]  可折叠          │  ← 详情区
│  [测试详情]               可折叠          │
│  [产品预览]               可折叠          │
└─────────────────────────────────────────┘
```

### 改动

| 文件 | 改动 |
|------|------|
| `src/components/DevCompleteDetailPanel.tsx` | 重构 overview TabsContent：(1) Verdict banner 保留但精简；(2) 新增 4 个指标卡片（评分+环形进度、测试通过率+进度条、变更统计、耗时），2x2 网格布局；(3) 时间轴从竖向改为横向单行，用连线+圆点+状态色；(4) 下方保留折叠详情区（Code Review 发现项、测试详情、产品预览），但用卡片容器包裹 |

### 关键视觉改进

- **指标卡片**：每个卡片有大字数字 + 小标签 + 进度可视化，bg-muted/30 圆角卡片
- **横向时间轴**：一行 5 步，圆点+连线，绿色/黄色/红色状态，悬停显示详情
- **发现项卡片化**：每个 finding 独立小卡片，左侧色条标识严重级别
- **更大留白**：section 间距从 space-y-0 改为 space-y-6，呼吸感更强
